# 0.1 ‚Äì What is JavaScript?

## Definition
JavaScript (JS) is a **high-level**, **interpreted**, and **dynamic programming** language
used to create **interactive effects** in mainstream **web browsers**. 
It runs **both in browsers and on servers (Node.js)**.

- **Actions:** Web page ko interactive banata
hai.

- **Updates:** Page ko reload kiye bina content
update kar sakte ho.

- **Events:** User ke actions pe response deta
hai.

- **Data:** Server se data fetch aur send karne
mein madad karta hai.

---

## Key Characteristics
- **High-level** ‚Üí Handles low-level details automatically (like memory).  
- **Interpreted** ‚Üí Runs line-by-line without compilation.  
- **Dynamically typed** ‚Üí No need to define variable types explicitly.  
- **Prototype-based OOP** ‚Üí Uses prototypes instead of classical classes.  
- **Event-driven** ‚Üí Executes code in response to user or system events.

---

## Why JavaScript?
- The **only language that runs natively in all browsers**.  
- Used for both **Frontend (React, Vue)** and **Backend (Node.js, Express)**.  
- Powers **modern web apps**, **mobile apps (React Native)**, and **desktop apps (Electron)**.  
- Has a massive ecosystem and community (NPM).

## üîß JavaScript Engine 
1. You write code in a `.js` file.  
2. Browser or Node.js interprets it.  
3. The **JS Engine** (like Chrome‚Äôs V8) converts it into machine code.  
4. Code executes line-by-line in a **single-threaded** environment.

# JavaScript vs ECMAScript

1. **ECMAScript (ES)**

- **Definition:** ECMAScript ek **standard specification** hai jo **ECMA International** organization ne banayi hai.

- Ye batata hai ki **JavaScript ka basic syntax aur features kaise hone chahiye.**
- Har naye update ka ek **version** hota hai, jaise:
  - **ES5** ‚Üí 2009 mein aaya
  - **ES6 (ES2015)** ‚Üí Classes, Arrow Functions, Let/Const, Promises add kiye gaye.

  - **ES7, ES8‚Ä¶** ‚Üí Naye features add hote rahe

- Matlab, ECMAScript = **Rules & Guidelines.**

2. **JavaScript**
- **Definition:** JavaScript **ECMAScript specification ka implementation** hai.

- Browsers (Chrome, Firefox, Safari, Edge) **JavaScript engines** ka use karke ECMAScript ke rules
follow karte hain.

- JavaScript extra **features bhi deta hai** jo ECMAScript standard ka part nahi hote, jaise:

  - DOM Manipulation
  - Web APIs (fetch, localStorage, etc.)
  - Event Handling
  - Browser-specific functions

# JavaScript vs TypeScript

|Aspect | JavaScript |TypeScript|
| ----- | ---------- | -------- |
|**1. Definition** | Ek scripting language jo browsers mein run hoti hai. | JavaScript ka **superset** hai jo extra features deta hai.|
|**2. Typing** | **Dynamic typing** ‚Üí Type declare karna zaruri nahi. |**Static typing** ‚Üí Type declare kar sakte ho, safer hai. |
|**3. Compilation** | Direct run hota hai browser ya Node.js mein. | Pehle **JavaScript mein convert** hota hai, phir run hota hai. |
|**4. Error Checking** | Errors mostly runtime par milte hain. | Errors compile-time pe detect ho jate hain.|
|**5. Project Size** | Small to medium projects ke liye best.| Large-scale projects ke liye better.|
|**6. File Extension**|  `.js `| `.ts` |
